# Platform Maturity

Tracks the evolution of SAP App Factory across sprints. **Legend:** 🟢 Implemented · 🟡 Planned (a real backlog item and trigger condition exist) · ⚪ Not Started (no backlog item yet) · 🔒 Deferred (deliberately postponed past a stated trigger condition).

This document has two parts: the **Official Roadmap** (the approved, named sequence of sprints — the authoritative answer to "what are we building next and why," maintained jointly with [ROADMAP.md](ROADMAP.md)) and the **Capability Maturity Matrix** (a granular, capability-by-capability view of implementation status). Only Sprint 0 and Sprint 1 have real, ticket-numbered backlog items (SAF-1 through SAF-38) behind them as of this document's last update — Sprints 2 through 8 state real objectives and dependencies, projected honestly from the product vision and existing ADRs, not from backlog tickets that don't exist yet. Treat anything beyond Sprint 1 as directional, not committed, until each sprint is actually planned against [DEFINITION_OF_READY.md](DEFINITION_OF_READY.md).

## Narrative: platform evolution strategy

**Sprint 0 built the seams, not the product.** Every row scored 🟢 in Sprint 0 is a *mechanism* — a port, a contract test, one real adapter, one real plugin proving the pattern — never a business capability a customer would recognize. This is deliberate: [00-vision-and-principles.md](docs/architecture/00-vision-and-principles.md)'s explicit Sprint 0 non-goals ruled out real SAP generation logic entirely.

**Sprint 1 (Discovery Workspace) is the pivot from foundation to product**, and the first sprint the Engineering Planning Principles' vertical-slice discipline applies to: it must deliver one complete, user-visible capability (a requirement submitted and processed end to end), not more platform plumbing. Its horizontal work (the Temporal spike, plugin isolation, the CVE fix, the first live CI run) is included *only* because the vertical slice genuinely needs it this sprint — plugin isolation, specifically, is a hard prerequisite the moment any real generation logic is on the table, which Sprint 2 introduces.

**Sprints 2 through 4 form one arc: Documentation Factory → Local Application Factory → Quality Engineering.** Each depends on the previous sprint's real output as its own generation input — a captured requirement becomes a generated spec, a generated spec becomes a generated (runnable) application, a generated application gets generated tests and a demo. This sequencing is deliberate, not arbitrary: skipping ahead (generating tests before there's a real application to test, or a real application before there's a real spec to generate it from) would mean building against a placeholder, exactly the anti-pattern Sprint 0's own discipline avoided throughout.

**`generated-app-kit` (SAF-31, ADR-0019) is the one shared dependency every generation capability needs** — the seven ports (persistence, authentication, authorization, messaging, storage, SAP connectivity, external API) and the mock/Enterprise-adapter contract-test suite. Building any generation capability before it exists would mean each one reinventing its own mock adapters — exactly the duplication risk R20 in the risk register names.

**Sprint 5 (SAP Enterprise Integration) is the sprint where "Local POC" stops being the only real execution profile.** Real customer/design-partner deployment risk (credentials, real BTP connectivity, tenancy commitments) is deliberately isolated to this one sprint, after four sprints of proving the generation pipeline works safely against mocks first.

**Sprint 6 (AI Delivery Organization) and Sprint 7 (Enterprise Governance) mature the platform's own intelligence and audit posture** rather than adding new generation targets — multi-agent orchestration and full Digital Twin traceability are both explicitly gated on there being enough real generated artifacts and workflow runs from Sprints 1–5 to actually orchestrate and project, consistent with Sprint 0's "don't build infrastructure ahead of a real consumer" discipline.

**Sprint 8 (Marketplace and Platform Intelligence) is deliberately last** — opening the plugin model to third parties before the platform's own capability/plugin/governance model has matured through seven prior sprints would mean building a marketplace on a foundation still likely to change underneath it.

**No cell or roadmap row should be treated as a commitment beyond what its cited backlog item (SAF-NN) or stated dependency already supports.** Where a cell has no ticket number, that reflects the honest current state — a real Sprint 2+ planning pass, following [DEFINITION_OF_READY.md](DEFINITION_OF_READY.md), is what turns each row from a directional objective into a committed, ticket-numbered story.
