# Continuous Improvement Backlog

**Established:** 2026-07-17, from the VS-1 (Discovery Workspace) Engineering Retrospective. This is a living document — add to it at the close of every future Vertical Slice, don't recreate it. Every item below is grounded in a concrete finding from VS-1's actual, built code (file/line references given where useful), not a generic best-practice checklist applied blind.

## How to read this backlog

- **Category A — Immediate.** Small, safe, no-behavior-change engineering improvements. Complete before VS-002 implementation starts.
- **Category B — Sprint Improvement.** Real, scoped work items. Schedule into Sprint 1's remaining capacity or Sprint 2.
- **Category C — Strategic.** Requires an ADR before implementation — architectural, not tactical.
- **Category D — Future Product Idea.** Recorded for later product/roadmap planning. Not implemented, not scheduled.
