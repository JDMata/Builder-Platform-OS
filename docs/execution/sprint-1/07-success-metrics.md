# Sprint 1 Success Metrics

**Revised 2026-07-16** for the merged, single-slice structure. Defines how Sprint 1's success is measured at the Sprint Exit Gate ([PROJECT_PLAYBOOK.md](../../../PROJECT_PLAYBOOK.md)). Metrics are honest counts and pass/fail checks against this sprint's own real scope — not fabricated percentages against an idealized target no one committed to.

| Metric | Definition | Sprint 1 target |
|---|---|---|
| **Completed Vertical Slices** | Count of VS-1 meeting its full Definition of Done ([02-vertical-slice-catalog.md](02-vertical-slice-catalog.md)). | 1 of 1 — Sprint 1 now has a single, larger Vertical Slice rather than two smaller ones. |
| **Capability Coverage** | Count of capabilities introduced that are registered *and* resolved through a real `CapabilityResolverPort` adapter. | 1 of 1 (`structure-business-requirement`). |
| **Digital Twin Coverage** | **Now a real, measurable metric** (previously tracked as "Interim Traceability Coverage," since `context-digital-twin` didn't exist) — the proportion of this sprint's Digital Twin node/relationship types (`Project`, `Platform`, `ProjectType`, `Owner`, `CreationEvent`, and their relationships) that are actually written by a real `GraphStorePort` adapter, matching [ADR-0021](../../adr/0021-project-digital-twin-knowledge-graph.md)'s model. | 100% (5 of 5 node types written and queryable). |
| **Automated Test Coverage** | Every one of the 23 tasks in [02-vertical-slice-catalog.md](02-vertical-slice-catalog.md) has its test layer(s) from [05-test-strategy.md](05-test-strategy.md) passing in CI, plus the one required end-to-end demo. | 23 of 23 tasks with a passing, owned test layer; 1 of 1 end-to-end demo passing unattended. |
| **Documentation Coverage** | Every new/changed package has a current README; every architecture doc a change makes inaccurate is updated in the same PR. | 4 new package READMEs (`persistence-postgres/requirements`, `persistence-postgres/project`, `context-digital-twin`, the new `GraphStorePort` adapter); [02-domain-model.md](../../architecture/02-domain-model.md) and [16-project-digital-twin.md](../../architecture/16-project-digital-twin.md) updated. |
| **Telemetry Coverage** | Every task with a Telemetry requirement has real OTel spans/structured logs, verified against a live collector, not just present in code. | All Telemetry requirements in [02-vertical-slice-catalog.md](02-vertical-slice-catalog.md) traced — including, now, Digital Twin writes specifically. |
| **Workflow Coverage** | The Discovery `WorkflowDefinition` exercises every "happy path" `WorkflowRunStatus` at least once in the real demo; `failed`/`cancelled` covered by automated tests. | 4 of 4 happy-path statuses in the demo; `failed`/`cancelled` covered by Task 1.20's tests. |
| **Business Value Delivered** | Pass/fail check against the Sprint 1 mission: *"a user transforms a business idea into an approved software project through an AI-guided discovery experience."* | Pass — the single demonstration ([06-demonstration-plan.md](06-demonstration-plan.md)) proves the full mission statement end to end, now including Digital Twin evidence the original plan didn't require. |
| **Technical Debt Introduced** | Every new item logged, classified by severity, given a resolution timeline, per [TECHNICAL_DEBT_POLICY.md](../../../TECHNICAL_DEBT_POLICY.md). | Recorded honestly at the Sprint Exit Gate; anticipated candidate: the minimal `context-digital-twin` implementation's scope gap against its full Sprint-7-originally-planned feature set (search indexing, semantic embeddings, Governance node types) — explicitly not built now, tracked as remaining Sprint 7 scope, not silently forgotten. |
| **Technical Debt Removed** | Count of previously-accepted debt items closed this sprint. | 2 — `drizzle-orm` CVE (Task 1.2) and CI never run on a real runner (Task 1.23) — both carried from the Sprint 0 Exit Gate. |
| **Sprint Goal Completion** | Binary — did Sprint 1 achieve its stated exit criteria? | Pass/fail, assessed at the Sprint 1 Exit Gate against the mission statement directly. |

## New note this revision: Digital Twin scope discipline

Because this sprint now builds a real (if minimal) `context-digital-twin`, the Sprint Exit Gate must explicitly confirm **what was deliberately not built** — search indexing, semantic similarity queries, Governance-context node types (`Risk`/`Incident`/`Problem`/`Change`), and any node type beyond the five this slice needs — remains correctly scoped to Sprint 7 (SAF-34–37's remaining work), not silently expanded because "the Digital Twin now exists." A metric of "Digital Twin Coverage: 100%" refers only to this sprint's own five node types, never to the full eventual Digital Twin's scope — the same "no invented percentages against an undefined denominator" discipline the original metrics doc already established.
